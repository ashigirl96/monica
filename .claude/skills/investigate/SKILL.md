# investigate

バグや不具合の根本原因を、実データとコードの両面から特定する。

## 基本姿勢

- **推測ではなくデータで語る** — DB・ログ・git 履歴など、手元にある一次ソースを直接確認する
- **仮説を立てたら即検証** — コードを読んで「こうなるはず」と思ったら、実データで裏を取る
- **レイヤーを跨いで追跡する** — 呼び出し元 → ビジネスロジック → ストレージ層 → 実際の DB 値まで一気通貫で追う

## 調査の流れ

1. **実データの確認** — まず現象を正確に把握する。DB を直接クエリし、問題のレコードの全フィールドを確認する
2. **イベント/ログの時系列再構成** — 関連するイベントを時系列で並べ、状態遷移を手動でトレースする
3. **コードパスの特定** — 問題に関わる関数・SQL を読み、各パラメータがどの値になるか机上トレースする
4. **fix 済みかの確認** — ビルドバージョンと git log を突き合わせ、修正が本番に反映されているか確認する
5. **テストの検証** — 既存テストが本番のパスを再現しているか確認する（テストダブルと本番実装の乖離に注意）
